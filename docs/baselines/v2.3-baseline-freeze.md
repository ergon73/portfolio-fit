# Baseline Freeze: v2.3

## Scope
This baseline locks the behavior of the current CLI pipeline before v3.0 rollout.

## Frozen Interfaces
- Entry point: `enhanced_evaluate_portfolio.py`
- Primary package CLI flow: `portfolio_fit/cli.py`
- Core output files:
  - `portfolio_evaluation_{scope}.json`
  - `portfolio_report_{scope}.txt`
  - `portfolio_compare_{scope}.json` (when `--compare` is used)
  - `portfolio_compare_{scope}.txt` (when `--compare` is used)

## Frozen Scoring Contract
- 17 criteria, 6 blocks, normalized total score `0..50`.
- Each criterion has:
  - score field (`number | null`)
  - metadata in `criteria_meta` with `max_score/status/method/confidence/note`
- Unknown criteria do not inflate score and reduce `data_coverage_percent`.

## Frozen Config
- Runtime config file: `portfolio_fit/scoring_config.json`
- Calibration patches are generated separately and are not auto-applied.

## Validation Gates
- Contract checks: `validate_evaluation_contract.py`
- Formal schema source: `schemas/portfolio_evaluation.schema.json`
- CI quality gates: `ruff`, `black --check`, `mypy`, `unittest`, `py_compile`

## Change Policy
Any change affecting scoring semantics, output fields, or thresholds must:
- update schema and documentation,
- include migration note in `docs/migration-v2_3-to-v3_0.md`,
- include changelog entry in `CHANGELOG.md`.
